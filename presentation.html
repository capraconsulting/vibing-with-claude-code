Safety Controls

  • Esc — Stop Claude mid-generation
  • Reject changes — Deny file edits or commands
  • /rewind — Undo the last turn, try a different approach
  • /rename — Name your session for later
02 / first steps

Checkpointing

Claude auto-checkpoints before every edit — press Esc + Esc or /rewind:

ActionWhat it does
Restore code + conversationRevert both files and chat
Restore conversationRewind chat, keep current code
Restore codeRevert files, keep conversation
Summarize from hereCompress everything after this point (frees context)

Think of checkpoints as local undo — git is your permanent history

Sessions

Rule 1: Be Specific

// vague

Make the UI better

// specific

Add input validation to the recipe form:
- Serving count must be positive
- Show inline error in red below input
- Disable submit until all fields valid
03 / prompting

Rule 2: Break It Into Steps

// too much at once

Build user auth, logging,
charts, and social sharing

// just right

1. Add a form to log an exercise
   (name, sets, reps, weight)

3. Display today's list below the form

CLAUDE.md & Memory

Teaching Claude about your project — permanently

04 / claude.md

What is CLAUDE.md?

  • A file Claude reads at the start of every conversation
  • Your project's instruction manual
  • Like .editorconfig or .prettierrc — but for Claude
$ /init

Generates a CLAUDE.md by analyzing your project

04 / claude.md

What to Include

# Project Overview
Workout tracker with React frontend and Express API.

# Tech Stack
- React 18 + TypeScript, Vite, Tailwind CSS
- Express.js backend with SQLite

# Commands
- npm run dev — start dev server
- npm test — run tests

# Code Style
- Functional components with hooks
- Named exports, no default exports
04 / claude.md

CLAUDE.md Hierarchy

LocationScope
~/.claude/CLAUDE.mdAll your projects (personal prefs)
./CLAUDE.mdThis project
./src/CLAUDE.mdSubdirectory-specific

Files closer to the code take precedence

Section 05

MCP Servers

Give Claude new superpowers

05 / mcp

What is MCP?

You  <->  Claude Code  <->  MCP Server  <->  Your API / Service
  • MCP servers expose tools Claude can call
  • Claude reads tool descriptions and decides when to use them
  • You approve/deny each tool call
05 / mcp

Express Route → MCP Tool

Your Express API

app.get("/api/workouts", ...)
app.post("/api/workouts", ...)
app.get("/api/workouts/stats", ...)

MCP Server wraps each route

server.tool("list_workouts",
  "List all logged workouts...",
  async () => {
    const res = await fetch(`${API}/workouts`);
    return { content: [...] };
  }
);

name = what it does · description = when Claude should use it · handler = calls your API

05 / mcp

Why Build Your Own MCP Server?

Without MCP

Claude can read/write files and run commands

With your custom MCP server

"What was my heaviest deadlift?"
"Log a workout: 3x10 bench at 80kg"
"Compare my reading stats to last year"

Claude becomes an intelligent assistant for your data

05 / mcp

Good Tool Descriptions Matter

// bad

name: "getData"
description: "Gets data"

// good

name: "search_workouts"
description: "Search workouts by
date range, exercise name, or
muscle group. Returns exercises
with sets, reps, weight. Use when
user asks about training history."
05 / mcp

Playwright: Browser Automation

$ claude mcp add playwright -- npx -y @anthropic-ai/mcp-playwright@latest

Then ask Claude:

Open my app at localhost:3000 and take a screenshot.
Does the layout look correct?

Claude can open browsers, navigate, click, fill forms, and screenshot

05 / mcp
// hands-on — 10 min

Build & Add MCP Servers

  1. Ask Claude to build an MCP server for your API
  2. Register it: claude mcp add my-project -- npx tsx mcp-server/index.ts
  3. Add Playwright: claude mcp add playwright -- npx -y @anthropic-ai/mcp-playwright@latest
  4. Ask Claude a question about your data
  5. Ask Claude to open your app in a browser and check it

Skills, Plugins & Agent Teams

Customize, package, and scale

06 / skills

Skills: Custom Slash Commands

Create .claude/skills/review.md:

---
description: Review code for bugs and issues
user_invocable: true
---

Review recent changes. For each issue:
1. File and line number
2. Description
3. Suggested fix

Then just type: /review

06 / skills

Hooks: Automation Without AI

Shell commands that run at lifecycle points — no AI involved

// auto-format on edit

"PostToolUse": [{
  "matcher": "Write|Edit",
  "command": "npx prettier --write
    $CLAUDE_FILE_PATH"
}]

// play a sound when done (macOS)

"Notification": [{
  "matcher": "",
  "command": "afplay /System/Library/
    Sounds/Glass.aiff"
}]
06 / skills

Skills vs Hooks

SkillsHooks
WhatPrompt templatesShell scripts
HowAI processes the promptDeterministic execution
WhenYou type /commandAutomatic on events
ForReusable workflowsAuto-format, block, notify
06 / skills

Plugins: Package & Share

Bundle skills, hooks, agents, and MCP into one installable unit

Standalone

.claude/skills/review.md
→ /review

Personal, single-project

Plugin

my-plugin/.claude-plugin/plugin.json
my-plugin/skills/review/SKILL.md
→ /my-plugin:review

Shareable, namespaced, versioned

Test locally: claude --plugin-dir ./my-plugin
